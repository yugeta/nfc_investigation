# プラットフォーム別・カード別 検証マトリクス

## 1. 本フェーズの対象範囲
- 対象: ブラウザ（JavaScript + Web NFC）とPHP APIでのNFC読取/書込検証
- 非対象: iOS SafariでのWeb NFC実装・検証（本フェーズではNG）

注記:
- iOSはNFC機能が一部利用可能だが、今回の研究開発では対象外として計画する。

## 2. できること / できないこと（現時点）

### ブラウザ（Web NFC）でできること
- NFCタグへのNDEF書き込み（タグ種別・権限条件を満たす場合）
- NFCタグからのNDEF読み取り
- 読取・書込イベントのサーバーログ保存（PHP API経由）

### ブラウザ（Web NFC）で制約があること
- 低レベルAPDU通信やカード内部の機密領域アクセス
- すべてのICカードで自由に任意データを書き込むこと
- セキュアエレメント領域のデータ取得・更新
- カード発行元が制限する機密領域へのアクセス

### iOS Safari（本フェーズ）
- NG（Web NFC未対応のため実装・検証対象外）

## 2.1 ブラウザ別可否（目安）

| 環境 | Web NFC 読取 | Web NFC 書込 | 備考 |
|---|---|---|---|
| Android Chrome | 可能 | 可能 | 本PoCの主対象 |
| Androidの他ブラウザ | 条件付き | 条件付き | 実装エンジン依存 |
| iOS Safari | 不可 | 不可 | 本フェーズ対象外 |
| PCブラウザ | 原則不可 | 原則不可 | NFCハードと実装依存 |

## 3. 手持ち媒体ごとの検証可能性

| 媒体 | 読み取り | 書き込み | 備考 |
|---|---|---|---|
| Amazonで購入済みNFCタグ | 可能（高） | 可能（高） | 最初のPoC対象として最適。タグごとの仕様差（NTAG/MIFARE等）確認が必要。 |
| マイナンバーカード | 条件付きで可能 | 原則不可 | 公的個人認証に関わる領域は厳格制御。検証は法令・ガイドライン順守が前提。個人情報の取り扱いに注意。 |
| IC認証可能なカード式名刺 | 条件付きで可能 | 条件付き | カード規格と発行元仕様次第。NDEF公開領域があれば読取しやすい。 |

## 4. 推奨テストシナリオ

### シナリオA: 購入済みNFCタグ（最優先）
- NDEFテキスト書込
- NDEF URL書込
- 書込後の再読取整合性確認
- 連続100回読取の成功率・遅延測定

### シナリオB: マイナンバーカード
- まずはカード検出可否、技術種別判定、接続安定性を確認
- 取得データは最小化し、ログに機微情報を残さない
- 必要に応じて法務・コンプライアンス確認を事前実施

### シナリオC: ICカード式名刺
- タグ検出、NDEF有無、読取可否を確認
- 発行元仕様で許可された範囲のみ検証
- 書込可否はカード仕様書ベースで判定

## 5. 実装観点の最小ゴール
- ブラウザでタグ読取画面を実装
- 購入済みNFCタグへの書込/再読取を実装
- カード種別ごとに「成功・失敗・未対応」を判定表示
- 試験ログ（端末、OS、時刻、タグ種別、結果、処理時間）をPHP APIで保存

## 6. 注意事項
- マイナンバーカードを扱う場合は、法令・規約・ガイドラインに従う。
- 個人情報を含む可能性があるため、検証ログは匿名化/最小化する。
- 本番想定の前に、公開可能なテストデータのみでPoCを進める。
