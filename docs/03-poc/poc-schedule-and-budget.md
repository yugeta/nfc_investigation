# PoCスケジュールと概算予算モデル

## 1. 体制モデル（例）
- Androidエンジニア 1名
- バックエンドエンジニア 1名
- QAエンジニア 0.5名
- PM/テックリード 0.3名

## 2. 推奨6週間スケジュール
第1週:
- 要件ヒアリング
- 脅威モデルワークショップ
- PoC向けアーキテクチャ確定

第2週:
- Android NFCリーダー実装
- ペイロードパーサーとローカル検証

第3週:
- 検証APIとデータベーススキーマ
- E2E統合

第4週:
- セキュリティ強化（署名、nonce、リプレイ試験）
- 可観測性と監査ログ整備

第5週:
- 現地パイロット試験
- 端末固有課題の修正

第6週:
- KPI測定とレポート
- 本番ロードマップと見積り更新

## 3. Go/No-Go判定用KPI基準
- スキャン成功率
- 完了遅延のP50/P95
- 誤受理率（false accept rate）
- 誤拒否率（false reject rate）
- 運用エラー率

## 4. 予算見積りフレーム
次の式で試算する。
概算費用 = （開発工数[人月] × 単価）+ 試験端末/タグ調達費 + パイロット運用費

コストドライバー:
- サポート対象端末モデル数
- セキュリティ保証レベル
- バックエンド拡張性要件
- コンプライアンス/監査要件

## 5. PoC完了時の成果物
- プロトタイプのソースコード
- アーキテクチャおよびAPIドキュメント
- 再現可能シナリオ付き試験報告書
- リスク一覧と対策状況
- 本番フェーズ見積り
