# 開発手順（ブラウザ + PHP 版 NFC PoC）

## 1. 目的
この手順書は、ブラウザ（JavaScript/Web NFC）とPHPでNFCタグ認識PoCを、短期間で再現可能に進めるための実務手順を定義する。

## 2. 前提
- 実装言語はJavaScriptとPHPを主対象とする。
- ブラウザ検証はAndroidのWeb NFC対応ブラウザを主対象とする。
- iOS Safariは本フェーズ対象外。
- 検証媒体は以下を使用する。
  - 購入済みNFCタグ
  - マイナンバーカード
  - IC認証可能なカード式名刺

## 3. フェイズ別手順

### フェイズ0: 要件確定（1〜2日）
1. ユースケースを3つ以内に絞る（例: 認証、打刻、情報参照）。
2. 成功基準を数値化する（例: 読取成功率99%以上、中央値1.5秒以内）。
3. 対応端末の最小集合を決める（2〜3機種）。
4. ログ方針を決める（機微情報を保存しない）。

成果物:
- 要件メモ
- KPI定義
- 対応端末リスト

### フェイズ1: 開発環境準備（半日〜1日）
1. DockerとDocker Composeを準備する。
2. public 配下にWebフロントとAPIを配置する。
3. `docker compose up --build` でローカル起動する。
4. Android端末でNFC有効化と対象ブラウザ起動を確認する。

成果物:
- 起動可能なWeb+PHP環境
- NFC有効化チェック手順

### フェイズ2: 読み取り実装（2〜3日）
1. NDEFReader.scan() でタグ検出を実装する。
2. NDEFレコード（text/url等）の表示を実装する。
3. シリアル番号や読み取り時刻を表示する。
4. 失敗理由を分類表示する（未対応、権限拒否、読み取り失敗）。

成果物:
- タグ読み取り画面
- 読み取り結果ログ

### フェイズ3: 書き込み実装（1〜2日）
1. 購入済みNFCタグに対するNDEF書き込み処理を実装する（NDEFReader.write）。
2. 書き込み直後に再読取し、整合性を確認する。
3. 書込不可タグ・ロック済みタグのエラーハンドリングを実装する。

成果物:
- NDEF書き込み機能
- 書込後検証ログ

### フェイズ4: 媒体別検証（2〜3日）
1. 購入済みNFCタグ:
   - テキスト、URLの読取/書込を検証する。
2. マイナンバーカード:
   - ブラウザでの検出可否、応答傾向、エラー傾向を検証する。
   - 個人情報や機微データの取得を避ける。
3. ICカード式名刺:
   - NDEF有無、読取可否、仕様依存制約を確認する。

成果物:
- 媒体別テスト結果表
- 制約一覧

### フェイズ5: API連携（任意、2〜3日）
1. 読取結果をPHP APIへ送信する（POST JSON）。
2. 読取/書込イベントをNDJSONで保存する。
3. サーバー時刻付きで監査ログを残す。

成果物:
- PHPログAPI
- E2E動作確認ログ

### フェイズ6: 受け入れ判定（1日）
1. validation-checklist.mdに沿って判定する。
2. KPI実測値（成功率、P50/P95）を記録する。
3. Go/No-Goを決定し、次フェーズ課題を整理する。

成果物:
- 検証報告書
- Go/No-Go判定

## 4. 実装時の注意点
- UIDのみで信頼判定しない。
- 例外時に再試行手順を明示し、運用者が復旧できるUIにする。
- ログにはカード固有情報や個人情報を不要に残さない。
- マイナンバーカード検証は法令・ガイドライン順守を前提にする。
- Web NFC対応可否はブラウザ実装差があるため、端末ごとに結果を分けて記録する。

## 5. 最小マイルストーン
- M1: 購入済みNFCタグの読取成功
- M2: 購入済みNFCタグの書込と再読取成功
- M3: 3媒体の読取可否マトリクス完成
- M4: KPIを含むPoC報告完了
